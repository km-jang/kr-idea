# 🚀 풀 리빌드 기획안: "1인 퀀트 데스크" (UPGRADE-PLAN-MAX.md)

> 작성: 2026-07-16 · 전제: **기존 규칙·구조 전면 폐기 가능** (0원 원칙, 정적 파일, ZIP 배포 전부 협상 대상)
> 불변인 것 딱 두 가지 — "혼자 쓰는 도구"라는 본질, 그리고 "소유자는 비개발자"라는 현실.
> 그래서 이 기획의 제1 설계 제약은 돈이 아니라 **운영 가능성**이다. 돈으로 성능은 사도 단순함은 못 산다.

## 1. 무엇을 버리고 무엇을 얻나

| 버리는 것 (현행) | 얻는 것 (리빌드) |
|---|---|
| GitHub Actions 크론 (지연 1~2시간) | 상시 가동 서버 — **지연 0분, 장중 실시간 감시** |
| data.json 정적 파일 (21일 이력) | 시계열 DB — **수년치 분봉·일봉, 무제한 백테스트** |
| 하루 1~4회 스냅샷 | KIS 웹소켓 **실시간 체결·호가 스트림** |
| 단일 HTML + localStorage | 웹앱 — 로그인, 기기 간 자동 동기화, 실시간 차트 |
| ZIP 드래그 배포 (소유자 노동) | **git push 자동 배포** — 소유자는 승인만 |
| 규칙 기반 분석만 | **LLM 분석 데스크** — 공시 원문·증권사 리포트 PDF·뉴스 전문 해석 |
| 장 마감 후 아는 것 | **장중에 아는 것** (파수꾼 초 단위 알림) |

유지하는 것: 텔레그램 중심 알림, 빨강=상승/파랑=하락, "최종 판단은 사람" 원칙.

## 2. 목표 아키텍처

```
[실시간 소스]                    [분석 엔진]                      [인터페이스]
KIS 웹소켓(시세·체결) ─┐   ┌─ 팩터 점수기 (기존 4축 이식·확장)   ┌─ 웹앱 (실시간 차트·포트폴리오)
DART 공시 (분 단위)  ─┼─→ 수집 데몬 ─→ 시계열 DB ─→┼─ LLM 데스크 (공시원문·리포트PDF·뉴스) ─→ API 서버 ─→┼─ 텔레그램 2.0 (차트 이미지·즉시 알림)
뉴스·검색 트렌드     ─┘   └─ 백테스트 연구소 (매일 밤 자동 실험)  └─ [옵션] 반자동 주문 모듈
```

- 서버 1대에 전부 탑재(수집 데몬 + DB + 분석 + API). 분리·확장은 필요해질 때.
- GitHub은 코드 저장소·백업·예비 실행자로 강등 (현행 시스템은 6개월간 예비로 살려둠).

## 3. 모듈 상세

**M1 · 실시간 수집기** — 한국투자증권 KIS Developers 웹소켓/REST. 비용 0원(계좌만).
유니버스 500 → 2,000+ 전 종목 확장 가능(필터는 조회 시점에).

**M2 · 시계열 DB** — PostgreSQL+TimescaleDB 자가 호스팅(서버에 포함, 추가 0원).
일봉·분봉·수급·공시·뉴스·내 매매기록 전부 축적. "히스토리 2~3개월 기다림"이라는 개념 소멸 —
과거 데이터는 KRX/KIS에서 소급 적재.

**M3 · LLM 분석 데스크** — 매일: 공시 원문 전문 60건 + 뉴스 500건 + 증권사 리포트 PDF 요약.
하루 입력 ~200만 토큰, 출력 ~10만 토큰(월 22영업일 = 입력 44M/출력 2.2M) 기준 월비용:

| 모델 | 월 비용 | 용도 권장 |
|---|---|---|
| Gemini Flash-Lite | 약 7천원 | 1차 스크리닝(전량) |
| Gemini Flash | 약 2.6만원 | 표준 데스크 |
| Claude Haiku 4.5 | 약 7.7만원 | 판정 품질 중시 |
| Sonnet급 (심층) | 약 23만원 | 중요 공시·리포트만 선별 투입 권장 |

실전 최적은 **2단 구조**: Flash-Lite가 전량 1차 분류 → 중요한 것만 상위 모델 심층 분석 = 월 1~3만원으로 Sonnet급 품질의 90%.

**M4 · 백테스트 연구소** — 서버가 매일 밤 CONFIG 조합 수백 개를 과거 데이터로 자동 실험 →
"이번 주 최적 가중치" 리포트. 지금의 '전략 리그(가상 4전략)'가 '수천 전략 자동 탐색'으로 진화. 추가 비용 0원(서버 유휴시간).

**M5 · 웹앱** — Next.js. 로그인(소유자 1인), 포트폴리오 서버 저장(기기 동기화 자동 해결 — 구글시트 불필요),
실시간 웹소켓 차트. 호스팅은 서버에 포함 or Vercel 무료.

**M6 · 알림 2.0** — 차트 이미지 첨부, 장중 파수꾼(목표가·손절·급등락·공시 발생 **초 단위**),
조건 커스텀("외인 5일 연속 + 5일선 지지 뜨면 알려줘"를 자연어로 등록 → LLM이 조건식 변환).

**M7 · [옵션] 반자동 주문 모듈** — KIS 주문 API로 "제안 → 텔레그램 버튼 승인 → 집행".
완전 자동매매가 아닌 **사람 승인 필수** 구조 + 일일 금액 한도 + 연속 손실 서킷브레이커.
⚠️ 손익에 직결되는 모듈이라 별도 승인 게이트로 분리 — 나머지 전부 완성·신뢰 후 마지막에 결정.

## 4. 레벨별 비용 시나리오 (월간, 환율 1,400원/$·1,550원/€)

| 레벨 | 구성 | 서버 | LLM | 합계 |
|---|---|---|---|---|
| Lv1 부품 교체 | 기존 UPGRADE-PLAN (현행 구조 유지) | 0원 | ~5천원 | **~5천원** |
| **Lv2 엔진 교체 ★추천** | 상시서버(Hetzner 4GB) + DB + 실시간 수집 + LLM 2단 데스크 + 웹앱 + 파수꾼 | ~7천원 | 1~3만원 | **월 2~4만원** |
| Lv3 풀 데스크 | 8GB 서버 + Haiku/Sonnet 심층 + 리포트 PDF 전량 + 백테스트 상시 + 이중화 | ~1.5만원 | 8~23만원 | **월 10~25만원** |
| Lv4 기관 흉내 | FnGuide/QuantiWise급 데이터 단말 구독 | — | — | 월 수십~수백만원, **비추천** |

Lv4를 비추천하는 이유: 그 데이터가 주는 우위는 기관들끼리의 속도 경쟁 영역이고,
개인 스윙·중기 투자의 수익원(판단력·인내)과 무관. Lv2→Lv3 차이도 대부분 "LLM을 얼마나 깊게 쓰나"뿐.

## 5. 이행 전략 — 가동 중 갈아타기 (Strangler 방식)

```
R0 (상시)   : 현행 무료 시스템 그대로 가동 = 안전망. 폐기하지 않는다.
R1 (1~2주)  : 서버 개설 → DB·수집 데몬 구축 → 기존 수집과 병행하며 데이터 일치 검증
R2 (2주)    : 분석 엔진 이식 + LLM 데스크 가동 → 웹앱 알파 → 기존 대시보드와 나란히 사용
R3 (1~2주)  : 알림 이관 → 2주 무사고 확인 → GitHub 크론을 예비로 강등 (이중화 완성)
R4 (이후)   : 백테스트 연구소 → 조건 알림 자연어 등록 → (별도 결정) M7 주문 모듈
롤백 규칙   : 어느 단계든 문제 시 스위치 하나로 현행 시스템 복귀 (6개월 보존)
```

개발 자체는 AI가 전담(개발비 0원). 소유자 액션: 서버 결제 카드 등록, KIS 계좌 개설, 단계별 승인.
배포는 git push 자동화로 전환 — **ZIP 드래그 노동은 R2에서 졸업**.

## 6. 정직한 리스크 평가

1. **최대 비용은 돈이 아니라 복잡도.** 고장 지점이 3배가 된다(서버·DB·데몬). 대응 설계:
   자가 치유(프로세스 자동 재시작) + 헬스체크 실패 시 텔레그램 + 장애 시 현행 시스템 자동 폴백 +
   "AI 정비 체계"(문제 발생 시 AI 세션이 서버에 접속해 직접 진단·수리하는 런북·권한 구성).
2. **LLM 과신 위험**: 분석은 참고, 점수의 뼈대는 재현 가능한 규칙 유지(이 원칙만은 리빌드에서도 생존).
3. **M7(주문)은 손실 직결**: 승인 버튼·한도·서킷브레이커가 있어도 리스크는 소유자 몫.
   투자 손익에 대한 도구의 책임은 없다 — 이 문서는 투자 조언이 아니라 도구 설계다.
4. **정직한 한계**: Lv2~3는 수익률을 직접 올려주지 않는다. 사는 것은 **시간(자동화)·즉시성(장중)·깊이(원문 분석)·실험력(백테스트)**이다.

## 7. 결론 및 추천

- **스위트스팟은 Lv2 (월 2~4만원)** — 현행 시스템의 모든 불만(지연·이력 부족·기기 동기화·장중 무지·ZIP 노동)이 한 번에 해소되는 지점.
- Lv3는 Lv2를 2~3개월 써보고 "LLM 심층 분석과 백테스트를 실제로 매일 쓰는지" 확인 후 증설.
- 착수 신호: 아무 세션에서 **"UPGRADE-PLAN-MAX의 R1 진행해줘"** — 서버 견적·가입 안내부터 클릭 단위로 시작.
- 병행 원칙: 이 리빌드가 진행되는 동안에도 M2 튜닝 등 현행 시스템 개선은 계속 (안전망의 품질 유지).
